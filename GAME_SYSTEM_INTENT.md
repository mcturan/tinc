# GAME_SYSTEM_INTENT.md

STATUS: CRITICAL REFERENCE
TYPE: INTENT (NON-OPTIMIZED, NON-COMPRESSED)

---

# CORE IDEA

System must include elements that create attachment, curiosity, and repeated return.

Not forced.

Not annoying.

Natural pull.

---

# HUMAN PSYCHOLOGY BASIS

* Humans age physically, not mentally
* Even elderly people enjoy simple games
* Watching play / interacting with play is deeply rooted behavior
* Simple games can create strong attachment (example: offline dinosaur game)

---

# DESIGN PHILOSOPHY

We are NOT reinventing games.

We are:

* taking proven simple mechanics
* recontextualizing them inside the system

---

# GOAL

Create a space where:

* a 15-year-old wants to enter
* a 50-year-old engineer is not disturbed

---

# BALANCE TARGET

* accessible for everyone
* but rewarding for skilled users

---

# EXPERIENCE TARGET

User should feel:

"I can do this"
"I am good at this"
"I want to try again"

---

# HOOK MECHANISM

Short sessions:

* 10–60 seconds
* no friction
* immediate start

---

# NON-FORCED INTERACTION

System must NOT:

* push aggressively
* interrupt work
* behave like ads

---

# MASCOT VISION

Mascot is not a decoration.

Mascot is:

* emotionally reactive
* context aware
* behaves like a companion

---

Example idea:

"Pina is bored → asks user to go to library"

---

# LIBRARY IDEA (PNOT)

Environment-based interaction:

* user enters a space
* performs a short activity
* exits quickly

---

Important:

This must NOT be a single game.

It must be:

a set of micro experiences inside the same environment

---

# MICRO GAME PHILOSOPHY

Same environment → different experiences

Example:

* find a book
* fix misplaced items
* recall memory
* follow hints

---

# EXPANSION POSSIBILITY

Library → lab → city → other environments

---

# WAVL GAME VISION

RF / signal based interactive system

---

Example scenario:

User placed in a location:

* chooses antenna
* chooses frequency
* adjusts power

Goal:

Reach another location

---

# IMPORTANT DISTINCTION

This is NOT just a game.

This is:

* simulation
* learning tool
* real-world reflection

---

# LONG TERM VISION

User should be able to:

* learn inside the system
* apply outside the system

---

# CRITICAL DESIGN RULE

Not fully realistic.

Not fully simplified.

Target:

Playable realism

---

# CONNECTION TO CORE PRODUCT

QRVEE core idea:

"I want my voice to be heard"

Game version:

"Can my signal reach?"

---

# FUTURE POSSIBILITY

Simulation becomes:

* real planning tool
* decision support system

---

# XP & PROGRESSION IDEA

Users should:

* feel growth
* improve skills
* unlock complexity

---

BUT:

No fake rewards.

No spam XP.

---

# ADDICTION MODEL (CONTROLLED)

Goal is NOT:

blind addiction

Goal is:

return motivation

---

# UI EXPECTATION

* smooth
* fast
* no loading feeling
* visually clean

---

# ANTI-CRINGE AWARENESS

System must avoid:

* childish tone
* noisy visuals
* casino-like overload

---

# DESIGN INSPIRATION (IMPLICIT)

* simple global games (Mario, Dino)
* but elevated in context

---

# FINAL INTENT

This system must feel like:

a natural extension of the platform

NOT:

a separate toy
